# Support

This document is for user-facing help and maintainer triage.

## Best Contact

- General support: `support@weballoon.app`
- Security disclosures: see [SECURITY.md](./SECURITY.md)

## Good Information To Include

When asking for help, please include:

- your platform and version
- the app version
- what you expected to happen
- what actually happened
- whether the issue is reproducible
- screenshots if the problem is visual

If the problem involves account linking, billing, or sync, include the email used for the flow.
Do not send passwords, session tokens, or private browsing data.

## Common Support Categories

### Install And Launch

Use this for:

- app will not open
- macOS Gatekeeper prompts
- Windows installer issues
- Linux AppImage launch failures

### Account Linking

Use this for:

- magic link does not complete
- device is not linking after email confirmation
- linked email looks wrong
- dashboard and desktop account behavior questions

### Billing And Plans

Use this for:

- plan not refreshing on desktop
- checkout or portal problems
- subscription mismatch after linking

### Sync

Use this for:

- app catalog did not sync
- workspace structure is missing
- “synced to cloud” seems inconsistent

Remember: browsing data, cookies, and logins are intentionally not included in sync.

### Diagnostics

Use this for:

- crashes
- stability issues
- update failures
- renderer freezes

If diagnostics sharing is enabled, sanitized reports may already be available to the admin
dashboard.

## Response Expectations

Support requests are handled as quickly as possible, but severity matters:

- security issues: highest priority
- billing and account access problems: high priority
- launch blockers and crashes: high priority
- design feedback and feature suggestions: normal priority
